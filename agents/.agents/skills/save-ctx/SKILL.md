# Save Context

Write a detailed context summary to a markdown file in the current working
directory:

`context-saves/context-<harness>-YYYY-MM-DD-HHMMSS.md`

Use the agent harness name for `<harness>`: `claude-code`, `codex`, or
`opencode`. If the harness is unclear, use a short lowercase identifier.
Create `context-saves/` if needed. Use the current local timestamp.

Start the file with a one-line origin header:

`> Generated by <harness> (<model>) on YYYY-MM-DD HH:MM`

Include these sections:

## Objective

The overall goal or task currently being worked on.

## Decisions Made

Technical decisions, design choices, and user preferences established during
the session. Include reasoning when it matters for resuming work.

## Changes Made

Every file created, modified, or deleted. For each, briefly describe what
changed and why. Include paths.

## Current State

The last completed step and the current codebase or workspace state relative to
the goal.

## Blockers & Issues

Problems encountered, unresolved errors, open questions, or required user input.

## Remaining Work

Concrete next steps to complete the objective. Include paths, function names,
commands, or implementation details when known.

## Key Context

Non-obvious details a fresh session needs: environment facts, quirks,
important file locations, conventions, and gotchas discovered during the work.

After writing the file, display the full contents of the saved file to the
user.
